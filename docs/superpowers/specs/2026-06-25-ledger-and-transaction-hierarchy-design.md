# 账本与收支信息层级调整

## 目标

让收支明细以“收支项目”为主要阅读信息，且让首页账本卡片一眼呈现收入、支出与包含初始余额的当前余额。

## 已确认的呈现方式

- 收支项目使用交易备注（`description`）并放在交易行上方，使用更大的正文字号。
- 分类显示在项目下方，使用较小的弱化文字；若交易没有备注，则分类仍作为主标题，以避免行标题为空。
- 标签保留在主标题同行，金额和原有编辑、删除交互保持不变。
- 创建和编辑账本时均提供“初始余额”数值输入；默认值为 `0`。
- 首页每个账本卡片仅展示三项财务指标：收入、支出、余额。余额按 `初始余额 + 收入 - 支出` 计算。

## 技术设计

- 不新增数据库迁移或 API 字段：`ledgers.initial_balance`、账本 POST/PUT API 和 `Ledger` 类型已具备该字段；本次将其接入侧边栏账本表单。
- `TransactionsView` 只调整现有行的 JSX 信息顺序和 Tailwind 字号/颜色，不修改交易列表请求、筛选、日期分组或金额计算。
- `DashboardView` 在每个账本统计结果中保留初始余额，基于已有统计收入、支出派生余额，替换记录数和分项笔数展示。

## 验证

- 为交易行标题回退和账本余额派生增加可独立测试的纯函数测试。
- 运行对应测试、TypeScript 检查与静默 lint。
- 启动本地应用，在桌面和移动宽度检查收支行的文本层级、账本初始余额输入及首页三指标卡片。
